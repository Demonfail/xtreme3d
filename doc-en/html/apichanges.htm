API changes


Xtreme3D 3.x is not 100% backward compatible with 2.x branch - there are several important changes, but in general functions names and their meaning remain unchanged. Virtually all of the old Xtreme3D 2.0 functionality was implemented, except for:

- querying position and normal of the last collision in DCE and FPSManager
- querying the number of rendered triangles
- portal culling. It is not clear how it worked (if worked at all) in original version of the engine, so this functionality was not implemented.

Features no longer present in Xtreme3D 3.x:
- LoadB3D (function worked not for all B3D files, now this format is loaded in usual way via FreeformCreate)
- ClearZones, ZonesVisible, CullZones, ZoneAddPortal, ZoneObjectInZone, ZoneGetCurrent, ZoneGetNew, ZoneAdjustSize, CreateZones, ZoneGetPortalCount, ZoneGetPortal
- TextSetColor, TextSetFont, TextSetText (now there are appropriate functions for each type of text)
- FreeformSetSecondTextureCoords, MeshOptimize, MeshSmoothFaces
- ObjectAlignToCamera, ObjectGetAtXY, ObjectSetDiffuseColor, ObjectStructureChanged, ObjectClueToBone
- ObjectIsClipped (function is the inverse of ObjectInFrustrum)
- MaterialGetFromLib
- MaterialAddTextureEx, MaterialTextureExClear, MaterialTextureExDelete (TextureEx system was rewritten for compatibility with shaders, and API was changed)
- ThorFXSetTargetAsObject
- DceGetGravity, DceVelocityCollided, DceGravityCollided, DceCountCollisions, DceGetCollidedObject, DceGetCollisionPosition, DceGetCollisionNormal
- FpsCountCollisions, FpsClearCollisions, FpsGetCollisionPosition, FpsGetCollisionNormal, FpsGetCollidedObject

We are interested in returning some of these functions to the engine. Therefore, if you have an appropriate experience, knowledge and desire to help us, please contact by e-mail (gecko0307@gmail.com) or via the forum (http://offtop.ru/xtreme3d).
