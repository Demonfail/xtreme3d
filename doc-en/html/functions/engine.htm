Engine

Class: -

Functions of the engine (engine), unlike the others, do not apply to any type of objects. Serve for the overall management of the Xtreme3D and set basic settings. The engine can be regarded as the most important and priority of all.

EngineCreate

Real EngineCreate =();
Initializes the Xtreme3D. After calling this function, you can use other functions of the engine. The engine initialization can occur only once for a one time, that is, you can not re-call this function until you EngineDestroy.

EngineDestroy

Real EngineDestroy =();
Destroys The Xtreme3D, freeing up memory from under all its resources. After calling this function, you can not use other functions of the engine until you EngineCreate.

EngineSetObjectsSorting

Real EngineSetObjectsSorting =( os as real );
Determines the sort mode (OS) for all objects. You have the following options: os
OsInherited = 0 - Legacy sort;
OsNone = 1 - no sort order;
OsRenderFarthestFirst = 2 - the first drawing objects with the maximum disposal from the camera (the default value);
OsRenderBlendedLast = 3 - opaque objects are not sorted and be the first to redrawn, transparent - the latest, and taking into account the depth;
OsRenderNearestFirst = 4 - the first drawing objects with minimal disposal from the camera.

EngineSetCulling

Real EngineSetCulling =( vc as real );
Determines the mode selection of visibility (vc) for all objects. You have the following options: vc
VcNone = 0 - lack of selection of visibility;
VcInherited = 1 - inherited the selection of visibility;
VcObjectBased = 2 - selection is made on the basis of the object: each object can be chosen depending on its position. The selection of the object does not affect the visibility of its child objects;
VcHierarchical = 3 - selection is carried out according to the principle of hierarchy: if the maternal object selected, all its child objects will also be invisible.
Depending on the structure of your scene, the most effective methods will vcObjectBased and vcHierarchical.

SetPakArchive

Real SetPakArchive =( filename as string );
Opens the compressed archive *.PAK file from which you can download resources engine. To create the archive requires PakEdit Utility.exe, which is part of the Xtreme3D SDK. It is recommended to create the archive and add this feature only when the final build the project.
Note that after calling this function, all of the resources without exception will be loaded the engine from this archive. Cancel this during the game is impossible.
Filename is the path to the archive.

Update

Real = Update( dt as real );
Commits the software update status of the engine (the calculation of collisions, animation, etc.). Usually called before ViewerRender and only once, even in the presence of several types.
Dt - step time in seconds. Changing this parameter it is possible to change the speed of the animation. The time step, as a rule, is inversely proportional to the number of frames per second (fps).

EngineSaveScene

Real EngineSaveScene =( filename as real );
Saves all scene objects to a file. The experimental feature, with some types of objects may not function properly.
Filename is the name of the file.

EngineLoadScene

Real EngineLoadScene =( filename as real );
Removes all scene objects and loads the new from the file. The experimental feature, with some types of objects may not function properly.
Filename is the name of the file.

EngineRootObject

Real EngineRootObject =( );
Returns the ID of the root object scene - that is, the object, whose descendants are objects with zero parent.
