Font & Text

Класс: TGLWindowsBitmapFont, TGLHUDText

Шрифт и текст. Xtreme3D поддерживает три типа текстовых объектов: плоский, объемный и экранный. Экранный текст отображается поверх отрендеренной картинки и применяется для представления некоей текстовой информации в игре: количество жизней или патронов, различных сообщений и пр. Плоский и объемный текст расположены внутри трехмерного пространства - объемный, в отличие от плоского, имеет толщину.
Все типы текста используют специальные шрифтовые объекты, хранящие изображения текстовых символов - букв, цифр и знаков препинания. В Xtreme3D поддерживается три вида шрифтов.
Шрифт-изображение (Bmpfont) представляет собой одну большую текстуру с символьной "сеткой", в которую заранее записаны изображения символов. Ее можно создать вручную в графическом редакторе, нарисовать символы любого цвета и сложности. Однако такой шрифт может содержать ограниченный диапазон символов - обычно только таблицу ASCII, которая включает латинский алфавит, цифры и стандартные знаки препинания. Теоретически, диапазон может быть расширен с использованием кодовых страниц ANSI, но это решение не сработает, если вам нужно выводить текст с символами из нескольких разных алфавитов.
Полноценно эта проблема решается только с использованием Юникода. В Xtreme3D поддержка Юникода реализована в виде шрифта FTfont - он создается из произвольного TTF-файла и по необходимости динамически подгружает любые символы, которые вы захотите вывести, хоть китайские иероглифы (если, конечно, они поддерживаются вашим шрифтом).
Единственный недостаток этого метода - необходимость передавать текст строго в кодировке UTF-8. Game Maker 8, к сожалению, не имеет встроенной поддержки UTF-8, и если вы попытаетесь просто вывести русский текст, напечатанный в редакторе кода GM, то ничего не получится - строка будет передана в кодировке ANSI. Справиться с этим можно двумя способами: во-первых, сконвертировать ANSI-строку в UTF-8 функцией FTConvertANSIToUTF8, а во-вторых, загрузить текст из файла, закодированного в UTF-8. Второе более предпочтительно, поскольку ANSI в любом случае не позволит вам вводить символы, не соответствующие кодовой странице - то есть, при использовании в приложении русского языка нельзя, например, вводить греческие буквы.
Если вы хотите выводить только ASCII (то есть, латиницу), то ничего конвертировать не надо - ASCII-символы в UTF-8 выглядят "как есть" и не требуют никакого специального кодирования.
Для использования шрифтов FTfont в папке с игрой должна быть библиотека freetype.dll (ищите ее в Xtreme3D SDK). Если вы не используете эту функциональность, то библиотека необязательна.
В целях обратной совместимости (и работы с объемным 3D-текстом) оставлен также третий вид шрифтов - системный (WindowsBitmapfont). Это, фактически, Bmpfont, который автоматически генерируется из указанного системного шрифта, установленного в Windows. Данный вид шрифтов также работает только с ANSI и, к тому же, не может загружать шрифты из произвольных TTF-файлов, поэтому для экранного текста вместо него рекомендуется использовать FTfont.
Обратите внимание, что Xtreme3D не распознает в строковых значениях символ # как переход на следующую строку. Вместо этого рекомендуется использовать тринадцатый символ ASCII - chr(13). Например:
HUDTextSetText(text, "line 1" + chr(13) + "line 2").

BmpfontCreate

real = BmpfontCreate( width,height,hspace,vspace,intx,inty,start,end as real );
Создает новый шрифт и возвращает его id.
width,height - ширина и высота символа в пикселях
hspace,vspace - промежуток между символами по горизонтали и вертикали в пикселях
inx,inty - координаты начала отсчета символов (крайняя верхняя левая точка)
start,end - номера первого и последнего символа. Для латиницы эти значения соответствуют 32 и 95.

BmpfontLoad

real = BmpfontLoad( font as real,material as string );
Загружает изображение шрифта из текстуры указанного Материала.
font - id шрифта
material - имя Материала.

FTInit

real = FTInit( );
Инициализирует шрифтовый движок FreeType, отвечающий за загрузку шрифтов формата TrueType (TTF). После вызова данной функции можно создавать шрифты FTfont. Для использования шрифтового движка в папке с игрой должна быть библиотека freetype.dll.

FTfontCreate

real = FTfontCreate( filename as string, height as real );
Создает новый TTF-шрифт и возвращает его id.
filename - имя файла TTF
height - кегль (размер) шрифта в пикселях.

FTConvertANSIToUTF8

string = FTConvertANSIToUTF8( text as string );
Конвертирует строку из кодировки ANSI в UTF-8 (Юникод) для использования с в объектах текста с TTF-шрифтами. Входная строка не модифицируется, функция создает в памяти новую строку и возвращает ее. Во избежание утечки памяти не рекомендуется вызывать эту функцию постоянно - сконвертируйте строку один раз в начале программы и используйте ее.
text - текст.

WindowsBitmapfontCreate

real = WindowsBitmapfontCreate( fontname as string, size,start,end as real );
Загружает системный шрифт Windows и возвращает его id.
fontname - название шрифта, например: 'Arial'
size - размер шрифта
start,end - номера первого и последнего символа. Для латиницы эти значения соответствуют 32 и 95.

FlatTextCreate

real = FlatTextCreate( font as real, string as string , parent as real );
Создает плоский трехмерный текст и возвращает его id.
font - id шрифта
string - текстовая строка для отображения
parent - id родителя для текстового объекта (0 - отсутствие родителя).

FlatTextSetFont

real = FlatTextSetFont( text,font as real );
Задает шрифт плоского текста.
font - id текста
font - id шрифта.

FlatTextSetColor

real = FlatTextSetColor( text,color,alpha as real );
Задает цвет плоского текста.
font - id текста
color - цвет
alpha - прозрачность.

FlatTextSetText

real = FlatTextSetText( text as real , string as string );
Задает текстовую строку для отображения плоским текстом.
text - id текстового объекта
string - текстовая строка.

HUDTextCreate

real = HUDTextCreate( font as real, string as string , parent as real );
Создает экранный текст и возвращает его id.
font - id шрифта
string - текстовая строка для отображения
parent - id родителя для текстового объекта (0 - отсутствие родителя).

HUDTextSetRotation

real = HUDTextSetRotation( text,angle as real );
Задает угол поворота экранного текста.
text - id текстового объекта
angle - угол поворота в градусах.

HUDTextSetFont

real = HUDTextSetFont( text,font as real );
Задает шрифт экранного текста.
font - id текста
font - id шрифта.

HUDTextSetColor

real = HUDTextSetColor( text,color,alpha as real );
Задает цвет экранного текста.
font - id текста
color - цвет
alpha - прозрачность.

HUDTextSetText

real = HUDTextSetText( text as real , string as string );
Задает текстовую строку для отображения экранным текстом.
text - id текстового объекта
string - текстовая строка.

SpaceTextCreate

real = SpaceTextCreate( font as real, string as string , extrusion,parent as real );
Создает объемный трехмерный текст и возвращает его id.
font - id шрифта. Поддерживается только WindowsBitmapfont
string - текстовая строка для отображения
extrusion - толщина текста
parent - id родителя для текстового объекта (0 - отсутствие родителя).

SpaceTextSetExtrusion

real = SpaceTextSetExtrusion( text,extrusion as real );
Задает толщину объемного текста.
text - id текстового объекта
extrusion - толщина текста.

SpaceTextSetFont

real = SpaceTextSetFont( text,font as real );
Задает шрифт объемного текста.
font - id текста
font - id шрифта.

SpaceTextSetText

real = SpaceTextSetText( text as real , string as string );
Задает текстовую строку для отображения объемным текстом.
text - id текстового объекта
string - текстовая строка.
